## 1. 개요

### 막둥이는 이런 프로그램입니다

- **AI가 결정, 사람은 감독**
  시장 분석과 매수·매도 판단은 AI가 합니다.
  사용자는 결과를 확인하고 필요할 때만 개입합니다.

- **한국 주식 전용**
  코스피·코스닥 종목을 대상으로 합니다.

- **개인 계좌 전용**
  키움증권 개인 계좌를 사용합니다. 법인 계좌·위탁 계좌는 불가합니다.

### 무엇이 다른가요

- **실시간 감시** — 보유 종목을 초 단위로 지켜보며 손절·익절 자동 실행
- **메신저 중심** — 복잡한 프로그램 창 없이 텔레그램 하나로 운영
- **자율 학습** — 매일 매매 결과를 스스로 복기해서 개선

## 2. 일상 운영

### 하루 흐름

```
  08:30 ────── 프로그램 시작 (자동 또는 수동)
  09:00 ────── 장 시작, AI 분석 활성화
  09:00~15:20  매매 진행 (주요 이벤트는 텔레그램 알림)
  15:30 ────── 장 마감, 일일 결산 자동 생성
  15:35 ────── 프로그램 자동 종료 (설정 가능)
```

### 시작 확인

프로그램이 정상 실행되면 텔레그램으로 시작 보고가 옵니다.

```
📋 Makdoongi 시작
  시각: 08:30 | 브로커: 연결됨 | 모드: 실전
```

이 메시지가 오지 않으면 프로그램이 제대로 실행되지 않은 것입니다.

### 종료 방법

- **자동 종료** — 설정된 시각(기본 15:35)에 스스로 꺼집니다.
- **수동 종료** — 실행 창을 닫거나 Ctrl+C.
- **안전 종료** — 먼저 `/emergency_stop`로 매매를 멈춘 뒤 종료 권장.

## 3. 텔레그램 명령

### 조회 명령 (비용 없음)

| 명령 | 한글 | 설명 |
|---|---|---|
| `/status` | - | 전체 현황 요약 |
| `/report` | 성과 | 매매 성과 요약 |
| `/positions` | 포지션 | 보유 종목 상세 |
| `/strategy` | 전략 | 전략별 성과 |
| `/account` | 계좌 | 계좌 건강도 |
| `/market` | 시장 | 시장 현황 |
| `/closing` | 결산 | 일일 결산 |
| `/cost` | 비용 | AI 사용 비용 현황 |
| `/history` | 이력 | 최근 매매 이력 |
| `/postmortem` | 복기 | 어제 복기 보고서 |
| `/weekly` | 주간 | 주간 복기 |
| `/monthly` | 월간 | 월간 복기 |

### AI 제어 명령

| 명령 | 설명 |
|---|---|
| `/analyze` | 즉시 시장 분석 실행 |
| `/auto_start` | AI 자율 매매 시작 |
| `/auto_stop` | AI 자율 매매 중지 |
| `/scalp_start` | 실시간 감시 시작 |
| `/scalp_stop` | 실시간 감시 중지 |

### 긴급 제어

| 명령 | 동작 | 사용 시점 |
|---|---|---|
| `/emergency_stop` | 모든 매매 즉시 중단 | 이상 감지, 급락 |
| `/close_all` | 보유 종목 전량 청산 | 최악 상황 (승인 필요) |
| `/resume` | 비상 해제, 다시 매매 | 정상 상태 복귀 후 |

### 시스템 명령

| 명령 | 설명 |
|---|---|
| `/reload` | 설정 새로고침 |
| `/help` | 전체 명령어 목록 |

### 자유 입력

슬래시 없이 그냥 말해도 AI가 답합니다.

```
오늘 삼성전자 어때?       → AI가 분석해서 답변
지금 돈 얼마 있어?          → 현재 잔고 조회
화면 스크린샷 찍어서 보여줘   → 프로그램 상태 캡처
```

## 4. 보고서 종류

### 일일 결산 (장 마감 후 자동)

- 오늘 거래 횟수, 수익/손실 합계
- 종목별 손익
- 내일 전략 제안

### 주간·월간 복기

- 주간/월간 누적 성과
- 승률, 평균 수익률
- 잘된 점 / 개선점 분석

### 매매 복기 (매 거래)

- 진입 이유 / 청산 이유
- AI의 당시 판단 근거
- 결과 대비 평가

## 5. 안전 장치

### 개별 종목 보호

- **1차 손절** — 설정 % 하락 시 AI 판단으로 청산
- **강제 손절** — 더 큰 하락 % 도달 시 무조건 청산
- **익절** — 설정 % 상승 시 AI 판단으로 청산
- **트레일링 스탑** — 고점 대비 일정 % 하락 시 자동 청산

### 계좌 보호

- **일일 거래 횟수 한도** — 과도한 매매 방지
- **예수금 비중 제한** — 한 종목에 몰빵 방지
- **일일 손실 상한** — 정해진 손실액에 도달하면 당일 매매 중단

### 장외 보호

- 장 시간(09:00~15:20) 외에는 매매하지 않습니다.
- 공휴일, 임시 휴장일 자동 인식.

## 6. AI 비용 관리

### 기본 구조

AI(Claude)를 사용하므로 비용이 발생합니다. 막둥이는 비용 관리를 위해:

- **일일 한도** — 하루 사용할 수 있는 최대 금액 설정 (기본 $5)
- **한도 초과 시 차단** — 초과하면 AI 호출이 자동 중단되고 알림
- **비용 최적화** — 간단한 판단은 규칙으로 처리, 복잡한 것만 AI에 맡김
- **모델 분리** — 상황에 맞는 모델 선택으로 비용 절감

### 비용 확인

```
/cost    →  오늘 $X.XX / $5.00 (XX%) 사용
```

한도의 80%를 넘으면 텔레그램으로 경고가 옵니다.

## 7. 자주 생기는 상황

### 알림이 안 와요

- 텔레그램 봇을 차단하지 않았는지 확인
- 프로그램이 실행 중인지 `/status`로 확인
- 네트워크 연결 상태 확인

### 매매가 안 돼요

- 장 시간 확인 (09:00~15:20)
- `/status`로 브로커 연결 상태 확인
- 예수금이 충분한지 확인
- `/emergency_stop`이 켜져 있는지 확인 → `/resume`으로 해제

### 손실이 커요

1. **즉시 멈추기**: `/emergency_stop`
2. **상황 파악**: `/positions`, `/account`
3. **필요 시 청산**: `/close_all` (신중하게)
4. **분석**: `/postmortem`으로 당일 복기 확인

### 프로그램이 꺼졌어요

- 보통은 자동 재시작됩니다.
- 재시작 알림이 오지 않으면 컴퓨터에서 직접 다시 실행.
- 실행 파일 이름과 위치는 운영자에게 확인.

## 8. 운영 권장사항

### 일상 체크포인트

- **아침**: 프로그램 시작 보고 확인
- **점심**: `/status` 한 번 체크
- **저녁**: 일일 결산 확인, 이상 있으면 `/postmortem` 상세 확인

### 매주 확인

- `/weekly`로 주간 성과 검토
- 손실이 누적되면 AI 전략 조정 검토 (운영자 상의)

### 안전 수칙

- 전 재산을 넣지 마세요. 손실 감당 가능한 금액만 운영.
- 처음에는 소액으로 테스트하면서 패턴 파악.
- 이상한 거래가 보이면 즉시 `/emergency_stop`.
- 정기적으로 `/cost`로 AI 비용도 확인.

### 장기 운영

- 분기별로 전략 성과 점검.
- 시장 환경이 급변할 때(금리 변동, 규제 등)는 일시 중지 고려.
- 백업이 자동으로 되지만, 중요한 기록은 별도 저장 권장.
