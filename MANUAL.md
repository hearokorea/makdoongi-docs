## 1. 개요

### 막둥이는 이런 프로그램입니다

- **AI가 결정, 사람은 감독**
  시장 분석과 매수·매도 판단은 AI가 합니다. 사용자는 결과를 확인하고 필요할 때만 개입합니다.
- **한국 주식 전용** — 코스피·코스닥 종목 대상
- **개인 계좌 전용** — 키움증권 개인 계좌. 법인/위탁 계좌는 불가.

### 무엇이 다른가요

- **실시간 감시** — 보유 종목을 초 단위로 지켜보며 손절·익절 자동 실행
- **메신저 중심** — 복잡한 프로그램 창 없이 텔레그램 하나로 운영
- **자율 학습** — 매일 매매 결과를 스스로 복기해서 개선
- **자기 인식** — 24시간 슬라이딩 윈도우로 자기 상태를 진단하고 판단에 반영

## 2. 일상 운영

### 하루 흐름

```
  08:30 ────── 프로그램 시작 (자동 또는 수동)
  09:00 ────── 장 시작, AI 분석 활성화
  09:00~15:20  매매 진행 (주요 이벤트는 텔레그램 알림)
  15:30 ────── 장 마감, 일일 결산 자동 생성
  15:35 ────── 프로그램 자동 종료 (설정 가능)
```

### 시작 확인

프로그램이 정상 실행되면 텔레그램으로 시작 보고가 옵니다.

```
📋 Makdoongi 시작
  시각: 08:30 | 브로커: 연결됨 | 모드: 실전
```

이 메시지가 오지 않으면 프로그램이 제대로 실행되지 않은 것입니다.

### 종료 방법

- **자동 종료** — 설정된 시각(기본 15:35)에 스스로 꺼집니다.
- **수동 종료** — 실행 창을 닫거나 Ctrl+C.
- **안전 종료** — 먼저 `/emergency_stop`로 매매를 멈춘 뒤 종료 권장.

## 4. 보고서와 복기

### 4-1. 일일 결산 (장 마감 후 자동)

- 오늘 거래 횟수, 수익/손실 합계
- 종목별 손익
- 내일 전략 제안
- 전략별 성과 카운트

### 4-2. 주간·월간 복기

- 주간/월간 누적 성과 · 승률 · 평균 수익률
- 잘된 점 / 개선점 분석

### 4-3. 매매 복기 (매 거래)

- 진입 이유 / 청산 이유
- AI 의 당시 판단 근거
- 결과 대비 평가

### 4-4. 즉시 복기 — 큰 손실 직후

일일 복기는 장 마감 후 하루 묶어서 수행되지만, **강제 손절이나 -3% 이상 손실**이
발생하면 체결 직후 즉시 원인 분석합니다.

**흐름**
1. 큰 손실 매도 체결 확인
2. 매수 당시 근거·보유 기간 자동 조회
3. 패턴 자동 분류 (RSI 과매수 / 추세 피크 / 주말 보유 / 약한 신호 / 강제 손절 등)
4. 최근 30일 유사 패턴 건수 집계
5. 패턴별 개선 제안 생성
6. 텔레그램 즉시 경고 전송

**알림 예시**
```
🔴 즉시 복기 000660 (-5.23%)
사유: 강제 손절 발동
매수 근거: RSI 68 + ADX 52 + 반도체 모멘텀 강세
패턴: RSI 과매수, 추세 피크, 강제 손절
최근 30일 유사: 8건
개선: RSI 60+ 매수 자제 / ADX 50+ 추세 피크 진입 주의 / 진입 기준 재점검
```

**특징**
- AI 호출 0 — 순수 규칙 기반. 비용 없음.
- 즉시성 — 체결 직후 1초 내 알림.
- 이후 매수 판단 시 패턴 누적이 자동 반영됩니다.

## 5. 자가 진화 시스템

막둥이의 핵심 — "**학습·자가 인식 5단계 + Meta-cognition**" 이 완전 폐쇄 루프로 작동.

### 5-1. 학습·자가 인식 5단계

"누적 → 복기 → 학습 → 배우기 → 깨우치기" + 자기 인식(Meta-cognition) — 매 단계가 실제 작동.

| 단계 | 설명 |
|---|---|
| 1. 누적 | 모든 매매와 이벤트를 영구 저장 |
| 2. 복기 | 매 거래 후 자동으로 원인 분석 (즉시/일일/주간/월간) |
| 3. 학습 | 손실·성공·오탐·자동 발견 4종 패턴 DB 축적 |
| 4. 배우기 | 과거 패턴 top3 를 매 AI 판단에 자동 주입 + 종목 스크리너 점수 반영 |
| 5. 깨우치기 | 감지된 문제에 맞춰 운영 파라미터 자율 조정 + 3일 효과 검증 + 자동 원복 |
| Meta-cognition | 24시간 슬라이딩 윈도우 자기 진단 (5단계) |

**피드백 루프 3중 완전 폐쇄**
1. 손실 → 복기 → 학습 DB → AI 프롬프트 경고 + 스크리너 페널티
2. 익절 → 시그니처 추출 → 학습 DB → 스크리너 보너스 점수
3. 오탐 → 오탐 감사 → 개선 제안 → 자율 튜닝 → 3일 검증

### 5-2. AI 판단 근거 — 과거·현재·미래 통합

AI 가 매 루프마다 "과거 + 현재 + 미래" 3대 시간축 데이터를 자동으로 받아 판단합니다.
도구 호출 생략 여부와 상관없이 시스템이 능동 주입합니다.

**과거**
- 손실 패턴 DB top3 (RSI 과매수 / 추세 피크 / 주말 보유 등 반복 실수)
- 종목별 이전 거래 성과
- 최근 복기 요약 (최근 2일)
- 연속 손절 일수 (2일+ 시 자동 경고)

**현재**
- 코스피/코스닥/VIX/환율 스냅샷
- 매크로 (Fed/BOK/수익률곡선/크립토/리스크)
- 실시간 뉴스 이벤트 시그널

**미래**
- 향후 N일 경제 캘린더 (High impact만)
- 실적 발표 예정 종목
- DART 공시 플래그
- 장 마감까지 남은 시간

200자 요약이 AI 프롬프트에 자동 주입:
```
🕰️ 📉손실패턴:RSI_과매수×5회 | 🔥연속손실2일 | 📊코스피6,313(+1.5%)국면:상승추세
 | 📰뉴스:약간매수(B2/S1) | ⏭임박:FOMC회의(04-28) | 💼실적예정3건 | ⏰마감까지3.2h
```

### 5-3. 시스템 자가 점검 (5분 주기)

매매 실적 복기는 `/postmortem`이 담당하지만, **매매가 아예 안 일어나는 이상 상황**은
별도 체계가 5분 주기로 자동 감지합니다.

**주요 감지 항목**
- 무거래 경보 (장중 3시간+ 가동했는데 실매수 0건) — 점심시간·안전장치 발동·포지션 만석은 자동 skip
- 도구 검증 실패율 상승
- 루프 정체 (초단타/중장기)
- 수집기 노쇠 (30시간+)
- 기술부채로 인한 매매 차단

**심각도별 액션**
- WARNING 🟡 — 텔레그램 알림 (같은 종류 10분 쿨다운)
- CRITICAL 🔴 — 텔레그램 + 수리 큐 자동 등록

### 5-4. 오탐 감사 (10분 주기)

시스템이 발령한 경보·판정이 **사후에 잘못된 것으로 드러나는 경우**를 감사하는 메타
레이어입니다.

**5종 오탐 감사**
- 매도 "실패" 판정 후 실제 체결 발견
- "외부 거래" 감지 주문이 실은 프로그램 자기 주문
- 무거래 경보 후 1시간 내 실제 매매 발생
- 도구 검증 실패율 만성적으로 높음
- 프로그램 정지 의심 경보 후 정상 복귀

3일 내 5회 이상 누적 시 개선 제안 자동 발행 → 자율 튜닝 대상 등록.

### 5-5. 항상성 컨트롤러 (5분 주기)

시스템이 스스로 "정상 상태"를 정의하고 이탈을 자동 감지·복귀시킵니다. 사람의 체온·혈압이
자동 조절되듯 포트폴리오/루프/브로커 상태가 목표 범위에서 벗어나면 자동으로 보수 모드
전환·신규 매수 차단·브로커 재연결을 수행합니다.

**감지 대상**

| 지표 | 이탈 시 자동 조치 |
|---|---|
| 매도 실패 연속 횟수 | 신규 매수 차단 + 즉시 복기 발동 |
| 포지션 비중 | 신규 매수 차단 |
| 일일 실현 손실 | 매수 차단 + 보수 모드 |
| 브로커 연결 | 자동 재연결 시도 |
| 루프 정체 | 워치독 트리거 + 경보 |

이탈 중인 지표는 AI 프롬프트에 자동 주입돼 AI 가 매 판단마다 자동 인지.
```
🔴 항상성 이탈: 매도실패·포지션비중 → 신규 매수 자동 차단/보수 모드
```

### 5-6. 성공 패턴 학습

익절 5% 이상 매도 시 그 거래의 시그니처(그룹/진입 시간대/RSI·ADX·MACD/신호강도)를
자동 추출해 저장. 스크리너는 신규 매수 후보 평가 시 과거 성공 시그니처와 유사도 매칭해
보너스 점수를 자동 부여합니다.

### 5-7. 자동 발견 손실 패턴

기존 8종 정의된 손실 패턴 외에, **통계적으로 반복되는 신규 손실 조합**을 시스템이 스스로
발견합니다. 3회 이상 반복 + 평균 -3% 이하 손실이면 자동 등록. AI 프롬프트에 자동 주입
되어 이후 판단에서 회피 대상이 됩니다.

```
🔬 자동 발견 손실 조합 top3 (반복성 높음):
  · AI-초단타 그룹 / 장마감 임박 진입 / 목·금요일 / 3시간+ 보유 → 5회, 평균 -4.2%
```

### 5-8. 자율 튜닝

승인 없이 자율적으로 운영 파라미터를 조정하는 안전 시스템. safety gate 통과 시에만 실행:

- 화이트리스트 16개 파라미터만 허용 (매매 핵심 값 보호)
- 변동폭 ±20% 이내, 24시간 쿨다운
- 3일 후 효과 측정 + 악화 시 자동 원복

**이벤트 기반 자동 트리거 예**

| 상황 | 자동 조정 |
|---|---|
| 연속 손실 감지 | AI 루프 주기 +20% (충동 매매 억제) |
| 매도 실패율 높음 | 매수 RSI 과매수 기준 +2p (보수화) |
| 도구 검증 오탐 제안 | 신호 임계 5% 완화 |

`/tune` 명령으로 현황·이력·안전 파라미터 조회.

### 5-9. 자기 상태 진단

24시간 슬라이딩 윈도우로 승률·손절율·연속손실·AI 신뢰도 추세·거래 빈도 변화를 계산하여
5단계 자기 판정을 내립니다.

| 판정 | 조건 |
|---|---|
| 🟢 excellent | 승률 60%+ 이면서 손절율 20% 미만 |
| 🔵 good | 승률 40~60% |
| 🟡 cautious | 승률 20~40% 또는 최근 추세 악화 |
| 🔴 struggling | 승률 20% 미만 또는 연속 손실 5회+ |
| 🟠 recovering | struggling 후 회복 징후 |

AI 프롬프트에 "🧠 24h 자기진단" 섹션 자동 주입 — "지금 나는 struggling 상태" 라는 자기
인식이 매 판단에 반영됩니다.

**매시간 자동 알림** — 판정 변화 시 또는 이상 상태 진입 시 텔레그램 전송.
`/selfstate` 로 즉시 수동 조회 가능.

## 6. 인프라와 안전

### 6-1. 직접 매수 자동 인식 (외부 거래 동기화)

HTS / 모바일앱 / 전화 주문 등 막둥이 외부에서 직접 매수·매도해도 자동으로 감지되어
프로그램에 반영됩니다.

**감지 방식**
- 실시간 — 체결 이벤트를 즉시 인식 (보통 1초 내)
- 주기 폴링 — 5분마다 계좌 잔고 변경 추가 확인 (이중 안전망)

**알림 예시**
```
🟢 외부 거래 감지 | 삼성전자 005930 | 5주 매수 @72,000원
※ HTS/앱/전화 주문으로 추정 — 프로그램이 자동 반영함
```

**자동 처리**
- 매수 → 새 포지션 추가 + 평단가 자동 계산
- 매도 → 기존 포지션에서 수량 감소 또는 전량 제거
- 거래 이력 자동 기록 (외부 거래 표시)

**프로그램 지연 체결 자동 기록**

막둥이가 낸 주문이 체결 확인을 받지 못해 "매도 실패"로 처리했지만 나중에 뒤늦게 체결이
들어오는 경우(네트워크 지연·거래소 혼잡), 이런 지연 체결도 자동 감지해 거래 이력에
반영합니다. 자기 주문인지는 주문 직전 기록한 의도와 체결 내용을 매칭해 구분하므로 "자기
주문이 외부 거래로 오분류"되는 일이 없습니다.

**유령 포지션 이중 안전장치**
- 매도 시도 시 브로커 실제 보유가 없고 미체결 주문도 없는 상태가 3회 반복되면 "유령
  포지션"으로 확정 → 메모리/파일 동시 제거 + 알림 1회
- 기존의 5회 반복 실패 알림 스팸 차단

### 6-2. 수리 큐 (비용 의식형)

크래시·코드 오류·기술 부채 중 AI 수리 비용이 클 것으로 예상되는 건은 자동으로 고치지
않고 **수리 큐에 누적**됩니다.

**누적 대상**
- 재시작 시 감지된 크래시
- 높은 심각도 기술 부채
- 핫리로드 실패 모듈
- 시작 시 모듈 초기화 실패

**자동 수리 임계** — 추정 비용 $0.10 초과 시 자동 수리 차단 → 큐 누적
HIGH 심각도만 즉시 텔레그램 알림, 나머지는 조용히 누적.

**명령어**
- `/repairs` — 대기 목록 조회
- `/repairs history [N]` — 처리 이력
- `/repairs today` — 오늘 처리 요약
- `/repair <id 6자리> fix` — AI 수리 승인
- `/repair <id 6자리> skip 메모` — 무시 처리

**자동 resolve** — 시스템 자가 점검이 5분마다 pending 항목을 검사해 근본 원인이 해결된
것을 자동으로 해소 처리.

### 6-3. 재시작 시 포지션 안전

막둥이를 재시작해도 기존 보유 종목이 실수로 매도되지 않습니다.

**다중 안전 구조**
- 자동 청산 기본 OFF (옵트인 필요)
- 그룹 메타 유실 방지 — 분류 안 된 포지션은 보호 그룹으로 자동 할당
- 저장 시 자동 보정 — 빈 그룹 감지하면 보호 그룹으로 보정 + 경고 로그

**부팅 복구 4중 안전망**
1. 미완료 액션 자동 재개 (24시간 이내) / 오래된 건 자동 거부
2. 놓친 복기/일일 카운터/야간 리포트 자동 실행
3. 브로커↔파일 3중 동기화 (유령/수량/복구)
4. 24시간 내 크래시 감지 + 무결성 감사 + 강제 재조정

### 6-4. 로그·메시지 아카이브

시스템이 생성하는 모든 기록(파일 로그 + 텔레그램 송수신)을 한곳에서 일원 관리합니다.
모든 중요 로그는 30일 보관되며 감지 규칙과 결합해 같은 사고 재발 시 빠른 진단이
가능합니다.

### 6-5. 데이터 정합성 자동 관리

코드 개선 때마다 저장 데이터 필드 의미가 바뀌는데 **레거시 값은 그대로 남아** 의미
불일치가 생기는 문제를, 시스템이 능동적으로 감지·교정합니다.

- 코드 새로고침 직후 자동 감사
- 포지션 그룹 ↔ 보유 기한 불일치 자동 교정
- 거래 이력 누락 필드 자동 보정

### 6-6. 자투리 포지션 관리

AI 가 예수금 부족이나 그룹 배분 말미에 **8천원~4만원대 자투리** 매수를 반복하던 문제를
양방향으로 해결.

**신규 매수 차단** — 건당 주문 금액이 하한(기본 50만원) 미만이면 자동 차단
**기존 자투리 자동 청산** — 본전(또는 사용자 지정 임계) 회복 시 자동 매도

**운영 예**
```
/set_config tiny_liquidation_enabled true          # 청산 모드 활성
/set_config tiny_liquidation_threshold_pct -0.5    # 0.5% 손실까지 허용
/set_config min_order_amount 300000                # 주문 하한 30만원으로 완화
/set_config min_order_amount 0                     # 주문 하한 비활성
```

## 7. 안전 장치

### 개별 종목 보호

- **1차 손절** — 설정 % 하락 시 AI 판단으로 청산
- **강제 손절** — 더 큰 하락 % 도달 시 무조건 청산
- **익절** — 설정 % 상승 시 AI 판단으로 청산
- **트레일링 스탑** — 고점 대비 일정 % 하락 시 자동 청산

### 계좌 보호

- **일일 거래 횟수 한도** — 과도한 매매 방지
- **예수금 비중 제한** — 한 종목에 몰빵 방지
- **일일 손실 상한** — 정해진 손실액에 도달하면 당일 매매 중단

### 장외 보호

- 장 시간(09:00~15:20) 외에는 매매하지 않습니다.
- 공휴일, 임시 휴장일 자동 인식.

## 8. AI 비용 관리

### 기본 구조

AI(Claude)를 사용하므로 비용이 발생합니다. 막둥이는 비용 관리를 위해:

- **일일 한도** — 하루 사용할 수 있는 최대 금액 설정 (기본 $5)
- **한도 초과 시 차단** — 초과하면 AI 호출이 자동 중단되고 알림
- **비용 최적화** — 간단한 판단은 규칙으로 처리, 복잡한 것만 AI 에 맡김
- **모델 분리** — 상황에 맞는 모델 자동 선택

### 자체 계산 극대화

"AI 는 판단만, 계산은 전부 코드가" 원칙으로 비용을 낮추면서 품질은 향상시킵니다.

- **기술지표 사전 계산** — RSI/MACD/ADX/볼린저/거래량/추세를 코드가 미리 계산해 표
  형식으로 AI 프롬프트에 주입
- **도구 정의 캐싱** — 반복 호출 시 토큰 재전송 최소화
- **규칙 엔진 확장** — 명확한 케이스는 AI 호출 없이 즉시 결정
- **동일 정보 중복 전송 제거** — 통합 컨텍스트 생성 후 여러 판단에 재사용

정보량은 오히려 증가하고, AI 호출 횟수와 재전송 토큰은 감소합니다.

### 비용 확인

```
/cost    →  오늘 $X.XX / $5.00 (XX%) 사용
```

한도의 80%를 넘으면 텔레그램으로 경고가 옵니다.

## 9. 자주 생기는 상황

### 알림이 안 와요

- 텔레그램 봇을 차단하지 않았는지 확인
- 프로그램이 실행 중인지 `/status` 로 확인
- 네트워크 연결 상태 확인

### 매매가 안 돼요

- 장 시간 확인 (09:00~15:20)
- `/status` 로 브로커 연결 상태 확인
- 예수금이 충분한지 확인
- `/emergency_stop` 이 켜져 있는지 확인 → `/resume` 으로 해제
- `/selfstate` 로 자기 진단 확인 (struggling / cautious 상태면 자동 보수화 중)

### 손실이 커요

1. 즉시 멈추기: `/emergency_stop`
2. 상황 파악: `/positions`, `/account`
3. 필요 시 청산: `/close_all` (신중하게)
4. 분석: `/postmortem` · `/selfstate` 로 당일 복기 확인

### 프로그램이 꺼졌어요

- 보통은 자동 재시작됩니다.
- 재시작 알림이 오지 않으면 컴퓨터에서 직접 다시 실행.
- 재시작 후 `/selfstate` 로 이전 상태 복원 확인.

## 10. 운영 권장사항

### 일상 체크포인트

- **아침** — 프로그램 시작 보고 확인
- **점심** — `/status` 한 번 체크
- **저녁** — 일일 결산 확인, 이상 있으면 `/postmortem` 상세 확인

### 매주 확인

- `/weekly` 주간 성과 검토
- 손실이 누적되면 AI 전략 조정 검토 (운영자 상의)

### 안전 수칙

- 전 재산을 넣지 마세요. 손실 감당 가능한 금액만 운영.
- 처음에는 소액으로 테스트하면서 패턴 파악.
- 이상한 거래가 보이면 즉시 `/emergency_stop`.
- 정기적으로 `/cost` 로 AI 비용도 확인.

### 장기 운영

- 분기별로 전략 성과 점검.
- 시장 환경이 급변할 때(금리 변동, 규제 등)는 일시 중지 고려.
- 백업이 자동으로 되지만, 중요한 기록은 별도 저장 권장.

## 11. 텔레그램 알림 형식

모든 알림은 모바일 한 화면에 한눈에 보이도록 컴팩트하게 설계되어 있습니다.

### 시스템 알림
```
ℹ️ 시스템 알림 | 16:07 | 핫리로드
설정 변경 감지 → 재로드 완료
```

### 매수/매도 실행
```
🔵 매수 | [중기] 삼성전자 005930 | 10주 × 70,000원 = 700,000원
📝 AI 매수 판단
```

### 청산 (익절/손절)
```
🔴 익절 | [중기] 삼성전자 005930 | 10주 | 70,000→72,000 🔴+2.9% +20,000원 | 15분
📝 익절 목표 도달
```

### AI 판단 보류
```
🤖 AI 판단 | 익절 보류 | 삼성전자 005930 | 70,000→72,000 🔴+2.9%
📝 추가 확인 필요 | 재판단 60초 | 강제손절 -12%
```

### 외부 거래 감지
```
🟢 외부 거래 감지 | 삼성전자 005930 | 5주 매수 @72,000원
```

### 자기 진단 (매시간)
```
🔴 시스템 자기진단: STRUGGLING
  · 24h 매매 20건 (매수 12/매도 3)
  · 승률 15% / 손절율 65%
  · 평균 손익 -2.30% / 연속손실 4
  · 추세: declining / 빈도: rising
```

### 색상 규칙 (한국 증시 표준)

- 🔴 빨강 = 수익 (양수)
- 🔵 파랑 = 손실 (음수)
- ⚪ 회색 = 변동 없음 (0)
