# 변경 이력

막둥이에 추가된 **기능의 등장 시점**을 주 단위로 정리합니다.
세부 구현·수치는 공개 대상이 아닙니다.

---

## 2026년 3월 2주차 (03-10 ~ 03-13) — 첫 릴리스

- AI 기반 자동 매매 시스템 시작
- 텔레그램 조회·명령 기본 골격

## 2026년 3월 3주차 (03-16 ~ 03-22) — 초기 안정화

- 스케줄 모드 (개장 전 브리핑 / 장 중 / 장 마감 후)
- 계좌번호 자동 매칭
- 워치독 도입 (크래시 자동 감지 + 자동 복구)
- AI 자동 코드 수정 (크래시 시 코드 분석·수정 후 재시작)
- 일일 브리핑 자동 생성
- 자가 수정 화이트리스트 확장 — 텔레그램에서 전체 코드 수정 가능
- 브로커 연결 자동 재시도

## 2026년 3월 4주차 (03-23 ~ 03-29) — 실시간 매매 · 하이브리드 판단

- 초실시간 매도 감시 (초 단위)
- 브로커 실보유 자동 동기화 — 누락 종목 자동 복구
- 트레일링 스탑
- 장 마감 전 전량 청산 기능
- 하이브리드 AI 매도 (익절 시점에 AI가 "보유 vs 매도" 결정)
- 1차 손절 시 AI 판단 추가 (HOLD 가능)
- 익절·손절 기준 상향 조정
- 전략별 자금 배분 동적 비율
- 핫 리로드 도입 — 재시작 없이 코드 변경 반영
- 통합 모니터링 블록
- 텔레그램 메시지 체계 완성

## 2026년 4월 1주차 (03-30 ~ 04-05) — 안정성 근본 보강

- 브로커 실보유 기준 동기화 근본 재구성
- 유령 포지션 근본 방지
- 매수 안전성 다중 안전장치
- 초고도 분석 도구
- 매매 엔진 구조 재정비

## 2026년 4월 2주차 (04-06 ~ 04-12) — 중장기 전략 · 비용 최적화

- 중장기 전략 체계 도입
- 통합 인텔리전스 수집기
- AI 호출 비용 극한 최적화 (규칙 엔진 + 사전 계산)
- 듀얼 루프 체계 (초단타 + 중장기)
- 뉴스 이벤트 실시간 감지 → 즉시 매매 판단
- 24시간 인텔리전스 수집 (장외 시간 포함)
- 수익률 극대화 모듈

## 2026년 4월 3주차 (04-13 ~ 04-19) — 복기 · 학습 · 자가 진화 기반

- 서브시스템 전면 정밀화
- 즉시 복기 / 일일·주간·월간 복기 자동화
- 반복 손실 패턴 자동 발견
- 기술 부채 자동 추적 체계
- 품질 인증 체계
- 축 체계 도입 — 기능 모듈화 및 책임 분리

## 2026년 4월 4주차 (04-20 ~ 04-26) — 유기체 진화 · 자가 감사 완성

**매도·체결 안정성**
- 매도 실패 반복 루프 근본 수정
- 체결 확인 지연 주문 자동 복구
- 자율 튜닝 (안전 범위 내 자동 조정 + 효과 측정 + 악화 시 자동 원복)
- 수리 큐 체계 (AI 수리 비용 의식형 자동 누적)

**자가 인식 · 항상성**
- 자기 상태 진단 (5단계 자기 판정)
- 항상성 컨트롤러 (이탈 시 기본값 복귀)
- 오탐 자동 감사 (경보 품질 자동 관리)
- 외부 거래 자동 인식·동기화 (HTS·앱·전화 주문)
- 자투리 매매 자동 차단
- 로그·메시지 30일 보존 아카이브
- 축 간 이벤트 버스 (실시간 축 간 소통)

**유기체 7단계**
- 기억 · 배우기 · 깨우치기 · 통찰 · 자가수정 · 항상성 · 자기진단
  7단계가 서로 맞물려 끊김 없이 순환

**자가 감사 + 일괄 버그 정비**
- 시스템 원칙 자가 감사 **18개 원칙**으로 확장 (기억·학습·UNKNOWN 누적·핫리로드 감시·이벤트 버스·비용 리셋·자가진단 정합성·유령 블록·복기 대칭·수리 큐·경보 오탐율·API 드리프트·Silent Fallback·모듈 로그 침묵·시세 건강성·출력 품질·의도-실행 괴리·반복 이상 로그)
- 기회손실 학습 (조기 익절 후 가격 추적으로 익절 기준 자동 보정)
- pending 주문 자동 검증 (체결 확인 지연 방어)
- 거래 기록 자동 보정 (프로그램 시작 시 모호한 기록 자동 복구)
- 공휴일·프리마켓 자동 인식 — 장외 시간 경보 오탐 차단
- AI 비용 한도 자정 자동 리셋
- 개장·재시작 직후 루프 정체 경보 오탐 차단
- 텔레그램 알림 한글화 및 중복 알림 자동 차단
- 네이버 페이지 구조 변경 대응 (거래대금 상위 데이터 파싱 복원)
- 축 간 이벤트 연결망 일괄 복구 (payload 표준화 · 누락된 구독 연결)

**시세 데이터 · 사용자 경험 · 자동 정리 (주 후반)**
- 시세 데이터 다중 소스 체계 (한 곳 실패 시 다음 소스 자동 전환)
- 외부 페이지 구조 변경 시 설정만으로 즉시 대응 가능 (코드 수정 불필요)
- 사용자 경험 감시 3축 신설
  - 같은 알림 반복 / 내부 용어 사용자 메시지 노출 자동 감지
  - AI 판단 반복 실패 키워드 자동 추적 → 근본 원인 조사 트리거
  - 반복 에러 로그 자동 인지
- 자기 참조 정합성 감시 (내부 함수·속성명 변경 시 자가 감지)
- 수리 큐 자동 해결 — 근본 원인 해결된 과거 경보는 재시작 시 자동 정리
- 경보 메시지 구조화 — "현상 · 진단 · 조치" 3단으로 즉시 행동 가능
