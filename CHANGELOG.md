- **자투리 매매 근절** — 건당 주문 금액 하한(기본 50만원) + 기존 자투리 포지션 본전 회복 시 자동 청산.
- **유령 포지션 자동 정리** — 실제 보유 없는 포지션 3회 시도 후 자동 제거 + 알림 1회 (기존 5회 반복 스팸 차단).

- **항상성 컨트롤러** — 포트폴리오/브로커 상태 이탈 시 자동 보수화

## 2026-04-23 — 체결 기록 완결 + 자기진단 정상화 + 기회손실 학습

### 새로운 기능
- **기회손실 자동 학습** — 익절 매도 후 30분 동안 가격 추적. 매도가 대비 추가로 크게 올랐다면 학습 저장소에 기록 → 일정 기간 반복되면 익절 기준을 스스로 상향. "너무 일찍 팔아서 아쉬웠던" 패턴을 시스템이 스스로 개선.
- **pending 주문 자동 정리** — 주문 발송 후 체결 확인이 오래 안 오면 5분마다 자동 교차검증(증권사 API). 15분 이상 미해결이면 강제 종결하여 학습 데이터 오염 방지.
- **한국 공휴일 달력** — 2026년 공휴일(15일) 반영 — 공휴일에는 장외 감지가 자동 skip (경보 오탐 차단).
- **프리마켓 모드** — 09:00 개장 전 별도 상태 인식 (거래 가능 시간 외 경보 필터링).
- **시스템 시각 이상 감지** — 시계가 과거로 되돌아가거나 큰 점프가 있을 때 자동 경보.

### 핵심 버그 수정
- **자정 이후 비용 한도 영구 차단 해소** — 전날 한도 초과로 호출이 막히면 자정이 지나도 리셋이 안 돌던 버그 해결 (이제 한도 체크 시점에 날짜 확인 → 자동 리셋).
- **승률 0% 표시 버그** — 오늘 거래가 있는데도 승률/손절율이 0으로 나오던 문제. 자기 주문이 "외부 거래"로 잘못 분류되어 집계에서 누락되던 연쇄 원인 전부 정리.
- **외부 거래 오탐 근절** — 체결 확인 지연으로 프로그램이 낸 주문이 "외부 거래"로 뜨던 문제 해결 (09:27 매도한 종목이 10:11에 외부거래 알림으로 뜨던 연쇄).
- **유령 포지션 재생성 루프** — 유령 정리 후 증권사 데이터 지연으로 다시 복구되어 매도 실패 반복되던 문제. 10분 블록리스트로 차단.
- **루프 정체 경보 오탐** — 09:00 개장 직후 10분 / 프로그램 재시작 직후 10분은 경보 자동 skip.
- **이상 알림 반복 스팸** — 같은 경보가 반복 뜨면 자동으로 쿨다운 연장 (6회 이상)하고 10회 누적 시 오탐 리포트 자동 생성.
- **"COM 복구" 알림 연속 스팸** — 10분 쿨다운 적용 (09:49/09:51/09:57 같은 연속 3회 스팸 해소).
- **거래대금 상위 데이터 폴백** — 네이버 페이지 이상으로 가격 0 반환 시 최근 10분 캐시로 자동 복구.

### 안정성 개선
- **체결 기록 완결성** — 주문 발송 직후 pending 선기록 → 성공/실패/체잔 지연 도착까지 모든 경로가 기록 유지. 학습 원료(거래 이력) 누락 차단.
- **매도 실패 재시도 점진 연장** — 매도 실패 시 무한 즉시 재시도 대신 실패 횟수에 따라 2→4→8→10분으로 대기 확대 (증권사 데이터 반영 시간 확보).
- **COM 불안정 시 유령 판정 보류** — 최근 5분간 TR 타임아웃 반복 또는 재연결 직후엔 "3회 미확인 유령" 확정을 유보 (실제 보유인데 조회 실패로 오판되던 케이스 차단).
- **체결 지연 도착 시 자동 복구** — 프로그램 주문이 뒤늦게 도착해도 해당 pending/failed 기록이 자동으로 filled로 복구 → 자기진단 승률 집계에 정상 포함.
- **체잔 side 불명 시 의도 역추론** — 증권사에서 side 정보를 생략해도 최근 60분 프로그램 주문 의도 기록으로 SELL/BUY 자동 판정.
- **회화 컨텍스트 연속성 강화** — 텔레그램 대화 맥락 보관 8회 → 20회 (이어지는 질문 답변 품질 향상).
- **텔레그램 중복 응답 차단** — message_id 기반 중복 필터.

### 개선
- 파일 읽기 도구에 범위 지정(start_line/end_line) 추가, 검색 도구(`grep_source`) 신설 — AI 자가수정 시 불필요한 토큰 소모 감소.
- 자가수정 실패 시 에러 응답에 유사 라인 힌트 제공.
- 25축 공통 방어 유틸 (재시도 백오프, 디스크 공간 가드, SQLite 적응형 연결, 실패 누적 추적) 도입.
- 축간 자동 복구 — 핸들러가 10회 연속 실패하면 자동 비활성화 + 복구 대기열 등록.
- 새 모듈도 재시작 없이 런타임 반영 (핫리로드 대상 확장).

### 시스템 원칙 자가 감사 (Self-Audit)
- **시스템 원칙 자가 감사 축** 신설 — 막둥이가 스스로 "내가 설계대로 동작하고 있나?"를 5분마다 자문
- 11개 생명체 원칙 자동 체크: 기억 유지 · 학습 원료 · UNKNOWN 누적 · 핫리로드 감시 범위 · 이벤트 버스 활성도 · 비용 자정 리셋 · 자가진단 정합성 · 유령 블록리스트 · 복기 대칭성 · 수리 큐 적체 · 경보 오탐율
- 위반 감지 시 자동으로 수리 큐 등록 + 텔레그램 알림 (1시간 쿨다운)
- 프로그램 헌법(CONSTITUTION) 정식 문서화 — 생명체 7원칙 + 축 체계 8조
- 거래 기록 자동 보정 — 프로그램 시작 시 지난 24시간 내 "UNKNOWN" 분류된 외부거래를 자기 주문으로 자동 복구
